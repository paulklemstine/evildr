## Specific Code Issues Found

### Issue 1: Notes Timeout Race Condition (Medium Priority)

In `game-loop.ts:649-658`:
```javascript
const updated = await Promise.race([
  this.pendingNotesPromise,
  new Promise<string>((_, rej) => setTimeout(() => rej('timeout'), 5000)),
])
```

**Problem:** 5 seconds is insufficient for notes generation on slow Gemini responses. The notes LLM call happens CONCURRENTLY with the player's interaction time (60-120 seconds per turn). If the Gemini API responds in 3-8 seconds (common), and the player takes 30+ seconds to respond, notes are READY before the next turn starts. But if the player responds FAST (< 5 seconds), the notes may not be ready and the timeout fires.

**Recommendation:** The timeout should be adaptive ‚Äî if `pendingNotesPromise` hasn't resolved after 5 seconds, start the main LLM call anyway but KEEP waiting for notes, applying them immediately when they resolve (even mid-turn). A better pattern:
```javascript
// Start notes await but don't block on it ‚Äî use what's available
const notesResult = await Promise.race([...]).catch(() => this.state.currentNotes)
// The notes resolve in background and state.currentNotes is updated regardless
```

### Issue 2: CONDITION_ENGAGEMENT Directive Not Persona-Linked (Medium Priority)

The `CONDITION_ENGAGEMENT` directive tells the LLM to profile the player, but it has no knowledge of the PLAYER'S actual condition unless the player discloses it in textfield inputs. The directive says "by turn 3, you MUST have a working hypothesis" ‚Äî but for a Bipolar II player who doesn't explicitly say "I have Bipolar II," the AI must infer from behavioral patterns.

**Problem:** The inference window is too short. T1 might show impulsivity (slider 10, chaotic choice), but many people pick bold/fast on T1. The hypothesis at T3 based on 2 turns of behavioral data is unreliable.

**Recommendation:** Add to notes template: `**Behavioral Signals:** [{turn: N, signal: "description", pattern_hypothesis: "..."}]` so the LLM accumulates observations explicitly rather than inferring fresh each turn.

### Issue 3: Endgame Directive Text Clash with Adventure Framing (Low Priority)

The `ENDGAME_DIRECTIVE` says: "Do NOT generate radio choices that imply continuation. Instead, offer reflective choices." But the `GEEMS_MAIN` template says radio choices should be "All action. All exciting. NEVER offer 'stop' or 'rest' or 'reflect.'"

This creates a direct contradiction on T15. The LLM may follow GEEMS_MAIN (last instruction seen) over `ENDGAME_DIRECTIVE`.

**Recommendation:** Move `ENDGAME_DIRECTIVE` to the VERY END of the prompt when injected (currently it appears before `PRE_GENERATION_CHECKLIST` in `buildTurnPrompt()`). It should override, not be overridden.

### Issue 4: Image Subliminal Text Not Verified in Turn Output (Low Priority)

The prompt demands: "The FIRST image element each turn MUST contain a short phrase (1-3 words) embedded naturally." There is no validation that the LLM includes this. In prior playtests, this has been inconsistent.

**Recommendation:** In `parseLLMResponse()` or a post-parse validation, check that the first element of type `image` has a value string containing quotes or something suggesting embedded text. Log a warning if missing (don't reject ‚Äî just surface for quality monitoring).

## Meta-Analysis: What GEEMS Does Well (When Working)

1. **Setting agnosticism:** The "throw into action" firstrun approach means the scenario can accidentally align with a player's fixation (food hall for an eating obsession player). When it works, it's magical.

2. **Textfield as primary diagnostic channel:** The mandatory textfield every turn is correct. Hypomanic players WANT to write. The food metaphors that bleed into non-food contexts are exactly what you'd hope to capture.

3. **The arc toward authenticity:** The competitive eating obsession as performance/avoidance ‚Üí real hunger as authentic desire is a natural therapeutic narrative that the game can discover organically when notes work.

4. **NPC as mirror:** Yuki as competitor-turned-ally functions perfectly as a therapeutic mirror (the person you most need to connect with is the person you've been treating as adversary). This is exactly the kind of insight GEEMS can produce.

## What GEEMS Still Struggles With

1. **Cold start persona mismatch:** T1-2 can't adapt to the player's condition because the player hasn't disclosed anything. The game's adventure scenarios might be food-irrelevant for an eating-obsessed player. The persona has to volunteer food metaphors themselves and hope the LLM picks them up in time.

2. **Bipolar II phase timing is invisible:** A hypomanic player and a euthymic player look identical in T1 behavioral data. The condition can only be inferred from textfield content, which the player controls. If the player doesn't volunteer "I feel like I could eat the world right now" the hypomania is invisible.

3. **Therapeutic vs. entertainment tension:** GEEMS's dopamine-maximizing adventure format sometimes works AGAINST therapeutic depth. The "raise stakes every turn" directive can interrupt the quieter moments that therapeutic breakthroughs require. The T4 valley turn in this simulation is where the best therapeutic work happened ‚Äî but the prompt pushes AGAINST valleys.
